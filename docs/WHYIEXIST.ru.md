# Почему mcp-kaiten существует

## Проблема

Kaiten — популярный инструмент управления проектами, но у существующих решений для интеграции с MCP есть компромиссы:

- В одних реализациях автоматически создаётся 246 инструментов по API, что перегружает контекст ИИ
- В других упор на инфраструктуру (логи, метрики, диагностику), но без учёта времени, чеклистов, тегов и вложений
- Иные реализации включают операции с git, но без повторов запросов, таймаутов и кэширования
- Полный жизненный цикл карточки при разумном количестве инструментов редко встречается среди альтернативных серверов

## Что даёт mcp-kaiten

### 1. Полный жизненный цикл карточки в 41 инструменте

Охватывает типичный сценарий работы с карточками Kaiten без перегрузки контекста ИИ:

**Карточки** — создание, чтение, обновление, удаление, поиск с 15+ фильтрами, списки по пространству или доске

**Подзадачи** — список дочерних карточек, привязка и отвязка подзадач

**Комментарии** — просмотр, создание, изменение и удаление комментариев к карточке

**Учёт времени** — полный набор операций со списаниями времени, выборка по пользователю или по карточке за любой период

**Пространства и доски** — навигация по структуре: пространства, доски, колонки, дорожки, типы карточек

**Теги** — список всех тегов, добавление и снятие тегов с карточек

**Чеклисты** — создание, просмотр и удаление чеклистов; добавление и обновление пунктов

**Вложения** — список, загрузка и удаление файлов карточки

**Пользовательские поля** — список доступных настраиваемых свойств пространства

**Пользователи** — текущий пользователь, список пользователей, роли

### 2. Сбалансированное число инструментов

41 инструмент достаточно для реальных задач и не захлёстывает контекст ИИ. У других реализаций с сопоставимым охватом предметной области может быть 246 инструментов (например, сгенерированных по полному API). У альтернативных серверов с похожей «инфраструктурной» направленностью часто около 26 инструментов, но без учёта времени, чеклистов, тегов, вложений и пользовательских полей.

### 3. Оптимизация ответов под ИИ

Вся цепочка «запрос — ответ» заточена под использование в ИИ:

- Из API запрашиваются только нужные данные, справочная информация кэшируется
- Четыре уровня детализации с упрощением структуры для разных сущностей
- Автоматическое усечение слишком больших ответов, чтобы не съедать контекст
- В описаниях инструментов — подсказки по сценариям и перекрёстные ссылки на связанные действия
- Компактный формат ответов экономит токены

### 4. Надёжность

- Автоматические повторы с экспоненциальной задержкой и джиттером при ошибках сервера, лимитах и сбоях сети
- Защита от дублирования изменений при повторных попытках после сбоев
- Настраиваемые таймауты запросов
- Защита от аварийного завершения при необработанных ошибках
- Автоматическое усечение слишком больших ответов
- Структурированные журналы с настраиваемым уровнем детализации
- Понятные сообщения об ошибках с подсказками, что делать дальше

### 5. Умное кэширование

Пространства, доски, колонки, дорожки, типы карточек, пользователи и теги кэшируются в памяти с настраиваемым временем жизни. Устаревшие данные отдаются сразу, обновление кэша идёт в фоне — вызов не ждёт завершения обновления.

### 6. Ограничение доступа

Настраиваемые через окружение ограничения: какие пространства и доски доступны ИИ — важно в мультикомандной среде, когда не всё нужно открывать модели.

### 7. Ресурсы и подсказки MCP

Встроенные ресурсы MCP дают быстрый доступ к структуре рабочей области (пространства, доски) без лишних вызовов инструментов. Шаблоны подсказок ведут по типовым сценариям: создание карточек, отчёты по времени, обзор досок.

### 8. Стандартная конфигурация

Переменные окружения с проверкой корректности при запуске. Неверная конфигурация даёт разборчивое сообщение об ошибке. Поддерживаются произвольные базовые URL для коробочных и частных установок.

### 9. Минимальные требования к окружению

Очень небольшой объём внешних зависимостей, быстрая установка, мгновенный старт и узкая поверхность для потенциальных угроз безопасности.

### 10. Расширенный поиск

Более 15 фильтров поиска: заголовок, владелец, доска, колонка, теги, даты, тип карточки, признак завершения, исключение архива и другие.
